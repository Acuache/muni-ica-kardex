# MCP — descripción

> Qué vas a encontrar en esta carpeta (portable). La lista real del proyecto está en `mcp.md`.

En `mcp.md` encontrarás los **servidores MCP disponibles en el proyecto**. Cada entrada describe:

- **Qué es / para qué sirve** — para qué se usa ese MCP.
- **Herramientas clave** — las tools principales que expone.
- **Cuándo usarlo** — los triggers concretos que lo hacen relevante.
- **Notas / configuración** — scope (proyecto/usuario), autenticación, variables y gotchas.

## Plantilla — añadir un MCP a `mcp.md`

```markdown
## <nombre-del-mcp>

**Qué es / para qué sirve.** (1–3 frases)

**Herramientas clave.** `tool_a`, `tool_b`, …

**Cuándo usarlo.** (triggers concretos)

**Notas / configuración.** (scope, auth, variables, gotchas)
```
